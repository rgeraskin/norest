## TL;DR

Один браузер для работы и личного — рано или поздно хаос с логинами. Workspaces (Arc, Zen) изолируют только cookies; extensions, history и настройки остаются общими. Полное разделение даёт связка: профили Chromium + browser picker (Velja на Mac) для роутинга внешних ссылок.

## Один браузер — общая помойка

Если использовать один браузер и для работы, и для личной жизни, то наступление хаоса с логинами в разные сервисы — лишь вопрос времени. Особенно, если работать в IT 🙂

Почтовые сервисы на одном домене, личный/рабочий Github, Cloudflare, да тысячи их...

Именно поэтому многие приходят к концепции Workspaces и/или к [Arc Browser](https://arc.net/) (ныне — [Dia](https://www.diabrowser.com/)), как к одному из первопроходцев. Каждый workspace имеет свой набор cookies, что решает проблему с логинами.

Я юзал [Zen Browser](https://zen-browser.app/) — более функциональный **OpenSource** аналог. Да сейчас уже много где есть воркспейсы. Но.

## Workspaces — не панацея

Рано или поздно появятся расширения, которые нужны только для работы/дома. Кроме того, было б круто иметь разные настройки. Например, speed dial, набор закладок. Да и историю браузинга.

Проблема воркспейсов в том, что остальное всё крутится в одном инстансе браузера, а в нем подобного глубокого разделения добиться сложновато ((

Есть и более существенный косяк: открытие ссылок из сторонних источников. В каком workspace открыть линк из терминала/Slack/Zoom? Чтоб это настроить, скорее всего, придется заморочиться с расширениями.

## Профили в Chromium

Суть в том, что можно запустить браузер с разными профилями и получить полное разделение дом/работа. Свой history, cookies, extensions. По изоляции — как два разных приложения.

Проще всего начать, просто склонировав свой профиль, а потом его донастроить: сбросить куки, history, расширения и т.п. Собрать Speed Dial.

В [Vivaldi](https://vivaldi.com/), например, надо создать профиль через клик на иконке своего текущего профиля в тулбаре (если ее нет — вытащить в тулбар), потом *Manage Profiles* => *Add Profile*.

> Через *Profile Management* в настройках я кнопку *Add Profile* в упор не вижу 😁

Затем:

```bash
# MacOS
# Если раньше профили не использовались,
# то имя исходного будет Default, а нового - Profile 1
cp -a /Users/<ПОЛЬЗОВАТЕЛЬ>/Library/Application\ Support/Vivaldi/Default/ /Users/<ПОЛЬЗОВАТЕЛЬ>/Library/Application\ Support/Vivaldi/Profile\ 1/
```

Открыть новое окно с нужным профилем можно кликом на иконку профиля в тулбаре. Свитчиться между окнами в маке удобно через *Cmd+`*, так как это окна одного приложения. Поработал — закрыл рабочее окно.

## Что с открытием ссылок снаружи?

Запустить браузер с нужным профилем можно простой командой типа `chromium --profile-directory="Profile 1"`. Чтоб не делать это вручную, нужно использовать прослойку в виде *browser picker* в качестве дефолтного в ОС браузера.

Как вариант, [Velja](https://sindresorhus.com/velja) для Mac. Сейчас он стал платным, но и бесплатных аналогов хватает ) В нем настраиваются правила, например, по url-prefix или приложению, согласно которым он открывает линки там, где надо. Дефолтный браузер делаем [Vivaldi](https://vivaldi.com/) с *Default* профилем, для правил — рабочий профиль.

![Модальное окно Edit Rule в Velja: правило «Slack, Miro, iTerm2, Zoom», открывается в профиле Work; в Source Apps добавлены zoom.us и iTerm](/assets/browser-profiles/2026-05-19-18-31-58-image.webp)

Бонусом Velja умеет открывать ссылки сразу в нужном приложении, минуя браузер. Например, линк на zoom call из календаря откроется сразу в зуме, а не отправит тебя в браузер => зум.

А если страдаешь browser hopping'ом, browser picker — это мастхэв, т.к. браузеры меняются парой кликов 🙂
